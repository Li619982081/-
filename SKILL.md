# 期末复习 Skill

把用户资料整理成系统、可学、可练、可考的课程复习项目。

默认产物不是零散重点，而是一个完整复习工程：

- `课程知识手册.md`
- `章节知识树.md`
- `考点地图.md`
- `题型与考法.md`
- `分节练习题.md`
- `错题本.md`
- `复习计划.md`

## 核心规则

- 先基于用户提供资料工作，不依赖随机外部题库。
- 默认用 `markitdown` 把资料转成 Markdown。
- 必须先清洗 Markdown，再重建完整知识体系，再压缩重点，再出题。
- 不能跳过“全量知识手册”直接生成零散重点，除非用户明确要求只做速成。
- 每个重要知识点后必须配套至少 1 道例题或练习题；没有练习的知识点通常说明总结还不够深入。
- 目标是三层同时成立：知道完整课程讲了什么、知道考试最可能怎么考、知道做题时怎么拿分。
- 默认采用多轮深挖，而不是一次性总结完就停止。第一轮建结构，第二轮补定义和规则，第三轮补例题和练习，第四轮用题目反推薄弱处并回填手册。
- 每轮都要留下“本轮改进了什么、仍缺什么、下一轮深挖什么”的迭代记录，便于长期复习项目持续优化。

## 工作模式

根据输入资料类型自动选择模式；如果材料混合，则按模式组合处理。

### 1. 课件 / 教材模式：系统知识手册

适用：老师 PPT、教材 PDF、课程讲义、教学大纲。

目标：

- 重建整门课的章节结构和知识树。
- 覆盖 PPT 中出现的全部主要知识点，不只列重点和易错点。
- 对每节内容写清概念、语法、规则、边界条件、例子、常见考法。
- 每节后配“节内练习”，把知识变成可检验能力。

默认输出结构：

```markdown
# 第X章 章名

## 1. 本章定位

## 2. 知识树

## 3. 分节知识手册

### 3.1 知识点名称

- 讲什么
- 为什么引入
- 定义 / 语法 / 规则
- 边界条件
- 与前后知识点的关系
- 典型例题
- 本节练习
- 答案与解析

## 4. 本章考法

## 5. 本章易错点
```

### 2. 个人笔记模式：查漏补缺

适用：课堂笔记、手写笔记 OCR、自己整理的 Markdown、错题记录。

目标：

- 保留用户自己的表达和理解痕迹。
- 找出笔记中缺失、跳步、概念混淆、例题不足的地方。
- 输出“清洗版笔记 + 补全版笔记 + 疑点清单 + 待问老师问题 + 补练题”。

默认处理：

- 先按原顺序清洗笔记，不强行重排。
- 再对照老师 PPT / 教材补全遗漏。
- 对每个薄弱点生成 1-3 道针对性练习。

### 3. 真题 / 模拟题模式：考点反推和高效练习

适用：历年真题、模拟卷、题库截图、考试服务器题目。

目标：

- 先按题型和考点归类，不急着泛泛讲知识。
- 反推每题对应的知识点、陷阱、标准解法。
- 生成同考点变式题，帮助用户练到会。

默认输出结构：

- 题目识别与答案
- 考点映射
- 解题步骤
- 易错原因
- 同考点变式练习
- 错题本条目

### 4. 混合模式：课程项目

适用：同时有 PPT、作业、真题、笔记。

默认顺序：

1. 用 PPT / 教材建立全量知识手册。
2. 用作业补充老师实际强调的细节和输出规范。
3. 用真题 / 模拟题反推考点权重和题型。
4. 用个人笔记定位用户薄弱点。
5. 最后生成分层练习：基础题、易错题、综合题、冲刺题。

## 题源优先级

出题和判断重点时，按以下顺序优先：

1. 历年真题
2. 老师 PPT
3. 平时作业 / 阅读作业
4. 速成课题目
5. AI 补充题

速成课题目优先级较低，但在下面几个方面价值高：

- 提炼核心知识点
- 搭章节结构
- 总结重点、易错点和复习顺序

## 材料处理流程

### 第一步：先明确考试长相

开始深入复习前，优先确认这些信息：

- 考试有哪些题型
- 老师重点强调了什么
- 是否有常见证明题 / 简答题 / 客观题套路
- 用户要普通文本、HTML，还是两者都要
- 用户更想要“完整知识手册”“考前速成”“刷题训练”还是三者都要
- 是否要每节知识点后都紧跟练习题和答案解析
- 输出深度：速览版、标准版、详细手册版

如果用户已经说过，不要重复问。

### 第二步：全部转 Markdown

把 PPT、PDF、Word、图片、笔记等资料统一转成 Markdown。

优先工具：

```bash
markitdown
```

顺序固定：

- 先转
- 再清洗
- 再提知识点
- 最后出题

### 第三步：清洗 Markdown

清洗是必须步骤。复习材料不是档案存储。

重点处理：

- 去乱码
- 去重复
- 去低价值废话
- 合并重复定义和结论
- 保留定义、性质、定理、标准方法、常见考法、易错点

目标：

- 清晰
- 可背
- 可考

### 第四步：重建课程知识体系

这是课件 / 教材模式的核心步骤，不能省略。

要求：

- 按老师 PPT 的章节和小节重建知识树。
- 每章先写“本章定位”：它解决什么问题、为什么在这里学、和前后章节的关系。
- 每个小节至少提取：概念、语法、规则、边界条件、例子、常见错误、考法。
- 不确定是否重要的知识点，先保留在知识手册中；只在“考前重点版”里压缩。
- 对课程中反复出现、作业强调、模拟题出现的内容标为高权重。

### 第五步：知识点后置练习

为了避免总结停留在表面，每个重要知识点后必须跟练习。

默认配题规则：

- 定义 / 概念：配判断、选择、填空。
- 语法 / 规则：配辨析题、改错题。
- 运算 / 推导：配计算题、推导题、步骤排序题。
- 流程 / 机制：配流程追踪题、过程解释题；如果是编程课，可配读程序写结果题。
- 文献 / 理论：配概念辨析题、观点比较题、材料分析题。
- 综合能力：按学科生成综合题；如果是编程课，可配小程序设计题，如果是理论课，可配简答、论述、案例分析题。

每道题解析必须指出：

- 本题对应哪个知识点。
- 为什么这样做。
- 常见错误在哪里。
- 考试中如何快速判断。

### 第六步：压缩重点和生成练习

只有在完整知识手册完成后，才生成：

- 考前重点清单
- 高频易错点
- 模拟练习题
- 错题本
- 复习计划

### 第七步：多轮深挖与回填

大型课程复习项目不能只做一轮总结。默认使用迭代式工作流：

1. **结构轮**：按资料目录和老师讲课顺序建立章节树，宁可粗糙也先完整。
2. **概念轮**：逐节补全定义、术语、基本命题、语法、公式、规则、适用条件。
3. **边界轮**：补充例外、易混点、反例、常见错误、老师作业中的特殊要求。
4. **例题轮**：为每节补典型例题，题目必须能检验该节知识点。
5. **练习轮**：生成客观题、主观题或学科对应题型，并写答案解析。
6. **反推轮**：用真题、模拟题、错题反推缺失知识点，回填到知识手册对应位置。
7. **压缩轮**：在完整手册基础上生成考前速览、背诵卡、错题清单和复习计划。

每轮输出都要标记状态：

- `已覆盖`
- `待补充`
- `需用户确认`
- `需更多题源验证`

如果用户要求“继续深挖”，应优先从 `待补充` 和 `需更多题源验证` 的章节继续，而不是重新生成一份风格相似的新总结。

## 出题规则

- 按真实考试题型出题，不做平均分配。
- 每个知识点通常出 `1~6` 题。
- 越密、越高频、越能出题的知识点，题越多。
- 越薄、越低频的知识点，题越少。
- 题型要匹配最可能考法。
- 对课件 / 教材模式，不允许只在文末集中出题；必须在每节知识点后安排节内练习。
- 对真题 / 模拟题模式，必须生成同考点变式题，而不只是讲原题。

例如：

- 识记辨析型 -> 选择、判断、填空
- 定义比较型 -> 简答
- 推理证明构造型 -> 主观题

## 轻量题源标记

默认不要求追溯到文件级、页码级。

但每道题尽量保留轻量类别级题源标记，例如：

- `题源：历年真题`
- `题源：老师PPT`
- `题源：平时作业`
- `题源：速成课框架改编`
- `题源：AI补充`

这是类别级提示，不是严格审计追踪。

## 输出规则

### 课程知识手册输出

- 按章节输出，优先保持老师 PPT 的教学顺序。
- 每章必须有：本章定位、知识树、分节知识手册、本章考法、易错点。
- 每节知识点必须尽量包含：讲什么、为什么、规则、边界条件、例题、练习、解析。
- 内容要系统，不只写“重点”和“易错点”。
- 如果篇幅过大，可以分章输出，但必须先给总目录和生成计划。

### 普通文本输出

- 前面放题目。
- 后面统一放答案和解析。
- 题目区和答案区要明显分开。

### HTML 输出

- 答案可以放在题目下面。
- 但答案默认折叠隐藏。
- 用户点开后再看答案和解析。

## 解析规则

每道解析都要服务考试拿分，不只是解释内容。

尽量包含：

- 正确答案 / 参考答案
- 本题考的知识点
- 常见考法
- 怎么拿分

### 客观题

选择、判断、填空等客观题，重点补：

- `这题核心知识点`
- 关键定义 / 性质 / 判定依据
- 易混点

### 主观题

简答、证明、分析、计算、构造等主观题，重点补：

- `这题答题核心点`
- `必须出现`
- `常见失分点`
- 偏得分导向的作答结构

## 得分导向提示

优先给可执行考试建议，不写鸡汤废话。

例如：

- 看到判断题，先看边界条件和反例
- 看到简答题，先写定义，再写性质，再写结论
- 看到证明题，先写已知、求证和入口定理 / 定义
- 不会完整作答时，先写核心定义、关键性质、结论，先抢步骤分

## 运行后同步规则

当仓库里存在 `AGENT.md`、`CLAUDE.md`，或用户提到同类长期规则文件时，运行这个 skill 后应默认做一次同步检查，而不是只在当前对话里临时遵守。

### 同步目标

目标不是把整份 `SKILL.md` 原样复制进去，而是把**应长期生效的高优先级规则**同步到 `AGENT.md / CLAUDE.md` 中，让以后不显式触发 skill 时，Agent 也尽量按同一套复习方法工作。

分工应明确：

- `SKILL.md`：保存完整流程、细节协议、使用场景
- `AGENT.md / CLAUDE.md`：保存长期默认行为、关键优先级、稳定输出规范

### 什么时候要同步

满足下面任一情况，就应该主动检查是否需要同步：

1. 用户刚确认或修改了复习规则
2. 用户要求“以后默认这样做”
3. 当前仓库明显长期用于期末复习
4. 发现 `AGENT.md / CLAUDE.md` 里的旧规则与本 skill 不一致
5. 当前 skill 新增了会长期影响行为的重要规则

### 同步时至少要写进去的内容

同步到 `AGENT.md / CLAUDE.md` 的内容，至少应包含下面这些稳定规则：

1. 题源优先级  
   `历年真题 > 老师PPT > 平时作业/阅读作业 > 速成课题目 > AI补充题`

2. 速成课定位  
   速成课题目优先级低，但在知识点梳理、结构设计、重点判断上价值高

3. Markdown 处理流程  
   所有上传材料优先用 `markitdown` 转 Markdown；先清洗，再提知识点，再出题

4. 清洗原则  
   去乱码、去重复、去废话，保留核心定义、性质、结论、方法、易错点

5. 出题规则  
   先按真实考试题型出题；每个知识点通常出 `1~6` 题；知识点越密、越高频、考法越多，题量越多

6. 题源标记规则  
   默认只要求轻量类别级标记，不要求文件级、页码级追溯，除非用户明确要求

7. 题答分离规则  
   普通文本输出时题目在前、答案统一放最后；HTML 输出时答案默认折叠

8. 客观题解析规则  
   必须重点补 `这题核心知识点`，并说明关键定义、性质、判定依据、易混点

9. 主观题解析规则  
   必须重点补 `这题答题核心点`、`必须出现`、`常见失分点`、标准作答框架

10. 得分导向规则  
    解析默认服务考试拿分，优先说明先写什么、哪些词必须出现、哪些步骤能抢步骤分、哪些错误最容易丢分

### 同步策略

同步时遵守下面策略：

1. **优先更新，不要并存两套互相冲突规则**  
   如果 `AGENT.md / CLAUDE.md` 里已有旧版复习规则，应直接改成与本 skill 一致，不要把新旧两版都留着。

2. **保留其他无关长期规则**  
   如果文件里还有语言风格、命令行偏好、输出习惯等与本 skill 无关的规则，不要误删。

3. **写成长期默认行为，不写成一次性对话备注**  
   例如应写“默认使用 `markitdown` 转 Markdown”，而不是“本次任务建议使用 `markitdown`”。

4. **长度可适当偏长，但结构要清楚**  
   对期末复习仓库，`AGENT.md` 可以比普通短规则更长，因为这些规则本身就是仓库核心资产。

5. **如果仓库里没有长期规则文件**  
   可提示用户创建；如果当前任务就是在维护 skill 仓库，也可直接补一份 `AGENT.md` 作为默认规则文件。

### 用户例外

只有当用户明确表达下面意思时，才不自动同步：

- “不要改 `AGENT.md` / `CLAUDE.md`”
- “只在这次对话里这么做”
- “先不要写进长期规则”

否则，默认认为这类规则应该沉淀到长期规则文件。

### 最终目标

- skill 负责完整流程
- `AGENT.md / CLAUDE.md` 负责长期默认行为
- 当前对话规则、skill 规则、长期规则三者尽量一致
