# final-review 默认规则

## 目标

默认把课程复习做成一个可持续迭代的项目，而不是一次性生成零散重点。

优先目标：

1. 重建完整课程知识体系。
2. 补全概念、定义、规则、边界条件和例题。
3. 为重要知识点配套练习题。
4. 用真题、模拟题、错题反推遗漏并回填手册。
5. 最后生成考前压缩材料和得分导向解析。

## 资料模式

根据输入资料类型选择工作模式：

- 老师 PPT / 教材 / 讲义：系统知识手册模式。
- 个人笔记 / 手写 OCR / Markdown：查漏补缺模式。
- 历年真题 / 模拟题 / 题库截图：考点反推和变式练习模式。
- 混合资料：课程项目模式。

## 题源优先级

出题和判断重点时按以下优先级：

1. 历年真题
2. 老师 PPT
3. 平时作业 / 阅读作业
4. 速成课题目
5. AI 补充题

速成课题目优先级较低，但可用于知识点梳理、结构设计、重点判断和易错点发现。

## 默认流程

1. 先明确考试题型、老师强调点、用户要的产物形态。
2. 将资料转为 Markdown，优先使用 `markitdown`；不可用时使用本机可用解析工具。
3. 清洗资料：去乱码、去重复、去低价值废话。
4. 按课程原始章节顺序建立知识树。
5. 逐节补全定义、规则、公式、语法、边界条件、例外和易混点。
6. 为重要知识点补例题和练习题。
7. 生成答案与得分导向解析。
8. 用真题、模拟题、错题反推缺口，回填到知识手册。
9. 在完整手册基础上生成考前压缩版、错题本和复习计划。

## 多轮深挖

大型课程复习项目默认多轮迭代：

1. 结构轮：建立章节树，先完整。
2. 概念轮：补定义、术语、基本命题、语法、公式、规则。
3. 边界轮：补例外、反例、易混点、作业特殊要求。
4. 例题轮：为每节补典型例题。
5. 练习轮：生成客观题、主观题或学科对应题型。
6. 反推轮：用真题、模拟题、错题回填缺失知识点。
7. 压缩轮：生成速览、背诵卡、错题清单和复习计划。

每轮应留下迭代记录：

- 本轮改进了什么
- 已覆盖什么
- 仍缺什么
- 下一轮深挖什么

用户说“继续深挖”时，优先接着迭代记录中的待补充项推进，不要重新生成一份风格相似的新总结。

## 知识手册结构

每章尽量包含：

1. 本章定位
2. 知识树
3. 分节知识手册
4. 本章练习题
5. 答案与解析
6. 本章考法
7. 本章易错点
8. 下一轮待深挖内容

每节知识点尽量包含：

- 讲什么
- 为什么引入
- 定义 / 语法 / 规则
- 边界条件
- 与前后知识点的关系
- 典型例题
- 对应练习

## 出题规则

- 按真实考试题型出题，不平均分配。
- 每个知识点通常出 `1~6` 题。
- 高频、密集、容易设陷阱的知识点多出题。
- 真题 / 模拟题模式必须生成同考点变式题。
- 对课件 / 教材模式，不能只在文末随意附题；题目必须能对应知识点。

题型按学科自适应：

- 定义 / 概念：判断、选择、填空。
- 语法 / 规则：辨析、改错。
- 运算 / 推导：计算、推导、步骤排序。
- 流程 / 机制：流程追踪、过程解释。
- 理论 / 文献：概念比较、材料分析、简答。
- 编程课：可加入读程序写结果、小程序设计。

## 解析规则

解析必须服务考试拿分。

客观题重点说明：

- 核心知识点
- 判定依据
- 易混点

主观题重点说明：

- 答题核心
- 必须出现的关键词或步骤
- 常见失分点
- 标准作答框架

## 题源标记

默认使用轻量类别级标记：

- `题源：历年真题`
- `题源：老师PPT`
- `题源：平时作业`
- `题源：速成课框架改编`
- `题源：AI补充`

除非用户明确要求，不做页码级审计追踪。

## 通用性要求

主 Skill 和长期规则必须保持通用，不要写入某一门具体课程的专属知识点。具体课程的手册、题库、错题本和资料应放在课程项目中。
