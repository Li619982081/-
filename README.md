# final-review

`final-review` 是一个面向大学课程期末复习的通用 Skill。它的目标不是生成一页零散重点，而是把用户提供的课程资料整理成一个可持续迭代的复习项目：

- 先重建完整课程知识体系
- 再补定义、规则、边界条件和例题
- 再按真实题型生成练习
- 最后压缩成考前重点和错题本

它适合这些场景：

- 老师 PPT / 教材 / 讲义：生成系统知识手册
- 个人笔记 / 手写 OCR：查漏补缺和补全
- 历年真题 / 模拟题：反推考点并生成变式练习
- 混合资料：建立长期复习项目

## 核心理念

### 1. 先完整，再重点

很多 AI 复习总结的问题是过早“提重点”，导致知识点割裂、深度不够。这个 Skill 默认先做全量知识手册，再生成考前压缩版。

### 2. 知识点必须能被题目检验

每个重要知识点后应配套例题或练习题。没有练习的知识点，通常说明总结还停留在表层。

### 3. 多轮深挖，而不是一次性总结

大型课程复习项目默认走多轮迭代：

1. 结构轮：建立章节树和知识树
2. 概念轮：补定义、术语、规则、公式、语法
3. 边界轮：补例外、反例、易混点、作业特殊要求
4. 例题轮：为每节补典型例题
5. 练习轮：生成题目、答案和解析
6. 反推轮：用真题、模拟题、错题回填手册
7. 压缩轮：生成考前速览、背诵卡、错题清单和复习计划

## 工作模式

### 课件 / 教材模式

适用：老师 PPT、教材 PDF、课程讲义、教学大纲。

目标：

- 重建整门课的章节结构和知识树
- 覆盖资料中出现的全部主要知识点
- 对每节写清概念、规则、边界条件、例子、常见考法
- 每节或每章配套练习题

### 个人笔记模式

适用：课堂笔记、手写笔记 OCR、自己整理的 Markdown、错题记录。

目标：

- 保留用户自己的理解痕迹
- 找出缺失、跳步、混淆和例题不足
- 输出清洗版笔记、补全版笔记、疑点清单、补练题

### 真题 / 模拟题模式

适用：历年真题、模拟卷、题库截图、考试服务器题目。

目标：

- 先按题型和考点归类
- 反推每题对应的知识点、陷阱、标准解法
- 生成同考点变式题
- 沉淀错题本

### 混合课程项目模式

适用：同时有 PPT、作业、笔记、真题或模拟题。

默认顺序：

1. 用 PPT / 教材建立全量知识手册
2. 用作业补充老师强调的细节
3. 用真题 / 模拟题反推考点权重
4. 用个人笔记定位薄弱点
5. 生成分层练习和考前压缩材料

## 题源优先级

出题和判断重点时，按以下顺序优先：

1. 历年真题
2. 老师 PPT
3. 平时作业 / 阅读作业
4. 速成课题目
5. AI 补充题

速成课题目优先级较低，但可用于搭结构、提炼重点和发现易错点。

## 材料处理流程

1. 明确考试长相和用户目标
2. 将 PPT、PDF、Word、图片、笔记等资料转成 Markdown
3. 清洗 Markdown：去乱码、去重复、去低价值废话
4. 重建课程知识体系
5. 为知识点配例题和练习
6. 生成答案、解析、考法、易错点
7. 用真题 / 模拟题 / 错题反推缺口并回填
8. 生成考前压缩版

优先转换工具：

```bash
markitdown
```

如果 `markitdown` 不可用，可使用本机可用的 PDF、PPTX、OCR 或文档解析工具替代。

## 默认产物

一个完整复习项目通常包含：

- `课程知识手册.md`
- `章节知识树.md`
- `考点地图.md`
- `题型与考法.md`
- `分节练习题.md`
- `错题本.md`
- `复习计划.md`
- `迭代记录.md`

## 输出原则

课程知识手册应按章节输出，每章尽量包含：

1. 本章定位
2. 知识树
3. 分节知识手册
4. 本章练习题
5. 答案与解析
6. 本章考法
7. 本章易错点
8. 下一轮待深挖内容

练习解析应服务考试拿分：

- 客观题说明核心知识点、判定依据、易混点
- 主观题说明答题核心、必须出现、常见失分点
- 计算 / 推导题说明步骤和关键转折
- 编程课可配读程序写结果题
- 理论课可配概念辨析、材料分析或简答题

## Contributing

欢迎根据真实复习实践改进这个 Skill，尤其欢迎：

- 更好的多轮深挖流程
- 更稳定的资料清洗策略
- 更通用的知识手册结构
- 更贴近不同学科的题型模板
- 更强的真题反推和错题回填协议

请避免把某一门具体课程的专属内容写入主 Skill。具体课程的资料、题目和知识点应放在课程项目中；主 Skill 应保持通用。
